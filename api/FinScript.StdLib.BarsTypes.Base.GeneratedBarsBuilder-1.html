OnDataPoint(DateTime, double, double, double, double, in Quantity, bool)

Receives the next portion of data The data can be represented as already generated bar, with formed open, high, low and close values, or it can be a tick with current price only The inheritors must handle both cases. In general, bar data can be introduced as 4 ticks, for open, high, low and close prices. But, if we have input data already presented as OHLC bar, it is rather inefficient to generate 4 calls. Thus, it was decided to make this handler support both data types: bar and tick When data is a bar (isBar is true), you have to process a open, high, low and close otherwise you can process close only

protected override void OnDataPoint(DateTime time, double open, double high, double low, double close, in Quantity volume, bool isBar)
