Stores date and time value and its time zone. It allows to identify the time globally, all round the world. Globally coordinated time allows to use times across the time zones. For example, the expiration time for the order can be set in local time zone, whereas the server works with times of EST zone, and this will not be a problem because the time will be automatically converted to the target time zone See more details about globally coordinated time:

Remarks

This type is recommended to use only when the context has no information about time zone. For example,it can be used as an input parameters for data queries. But if the timezone information is available, it is recommended to use DateTime. Let's consider for example Trade. Since it provides information about its time zone via TimeZone all its time values (EntryTime, ExitTime) can be stored in standard DateTime format TimeZone. If the conversion goes out of instrument scopes, then use PrimaryTimeZone

Pay attention, the type does not support comparison (IComparable, IComparable<T> because for comparison, both arguments should be converted to the same time zone (for example, UTC), but this may produce duplication due to daylight time saving feature, that will lead to exceptions in sorted lists, unpredictable results on sorting, etc

Consider this class just as a container for a time transfer between application blocks

DateTime

Returns the clock date and time (without information about UTC time shift)

public DateTime DateTime { get; }

Property Value

DateTime

Remarks

Be cautious about using this property. Since the DateTime does not contain information about UTC shift, a returned value may lead to unpredictable behavior, like incorrect comparison. For example, market times 04:00:00 (UTC+4) and 03:00:00 (UTC+3) are equal, but after extracting their "clock" times (04:00:00 and 03:00:00) values will not be equal anymore It is strongly recommended to use ToDateTime(TimeZoneInfo) instead to be sure you get the clock time exactly in the TZ you want

To convert the value to date and time of different time zone use ToDateTime(TimeZoneInfo)
