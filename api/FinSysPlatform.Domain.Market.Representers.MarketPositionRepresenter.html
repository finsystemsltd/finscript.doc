The formatter for MarketPosition. This class is commonly used to bind XAML views to property of MarketPosition with predefined formatting settings Using this class you should not care about how the trade action title really must look and should not use any converters for it Using the formatter in model is better than using custom converters in XAML, because 1. With formatter XAML notation looks much shorter and simpler. Especially, if you refer to the model property from several XAMLs 2. It gives you strong-typed compile-time checking of binding 3. Representer may be context-driven (for example, it may use global settings of the app) in future
