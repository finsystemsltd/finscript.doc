Remarks

Unlike the most other market instances, Instrument does not provide equality implementation, because there are several ways of comparing entities: 1. By key (IsSameKey(Instrument, Instrument?)) 2. By parameters (ParametersEqual(Instrument, bool)). Here are 2 variants as well: a. with updateable parameters b. without updateable parameters 3. By instance refs, IsThis(Instrument?, Instrument?)

PointSize

Indicates the minimal possible fraction of the price. Usually it is mirror of Precision For example, it Precision=2, Point size most likely will be 0.01 Do not confuse with TickSize, which describes the minimal possible price change. For example, point size 0.01, tick size = 0.05, which means that minimal price change is 5 points (pips)

Remarks

Remarks

Since instrument has 3 ways of comparison (by reference IsThis(Instrument?, Instrument?) , by properties ParametersEqual(Instrument, bool) and by key UniqueKey) it is highly recommended to use IsThis(Instrument?, Instrument?) method instead of standard Equals(object) to avoid ambiguity This method behaves like standard Equals(object) and still compares by reference, so, technically, it is equal to IsThis(Instrument?, Instrument?)

Operators

operator ==(Instrument?, Instrument?)

Two instruments can be compared by 3 different methods:

  1. By reference equality, see IsThis(Instrument?, Instrument?) or standard method ReferenceEquals(object, object)
  2. By field to field comparison, see ParametersEqual(Instrument, bool) method
  3. By UniqueKey comparison, see IsSameKey(Instrument, Instrument?) method To avoid ambiguity of equality, equality operator is defined as obsolete. Use specific methods instead
[Obsolete("Possible comparison ambiguity. In order to compare instrument instance references, use IsThis method, to compare by attributes, use  ParametersEqual method. If you need to compare only UniqueKey, then IsSameKey method, if you need to check for null, use is null")]
