Task (optionally, depends on provider) which executes a request May be null if the provider cannot guarantee the query is finite. Also, if provider has orders cached or this is simulation provider it may return CompletedTask The executing task never throws an exception. For indicating a problem it should return an exception, or null if it completes successfully

Remarks

Usually the logic of calling side tp process result is:

  1. Is task null? If yes, then we cannot get the moment when then operation will complete, exiting...
  2. Wait for task completion. If the task returns exception, then request failed, otherwise we can read refreshed entities (positions, executions etc)
