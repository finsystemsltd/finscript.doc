Class Trade

Namespace
FinSysPlatform.Domain.Market

FinSysPlatform.Domain.Market.dll

Represents entry+exit pair. Depending on broker, this pair can be virtual (calculated from real executions) For example, netting accounts afford to add new portions to existing position and close position partially etc. So, for netting account deal can consist of several entries and one exit For hedging accounts usually entry+exit pair usually represents real trade Attention: deal is always calculated from exit. Deal can have multiply entries (for netting accounts) but always has one exit

Examples

Example #1: Hedging account, EUR/USD, buy 0.2 lots at 1.10000, then exit 0.1 lots at 1.10100. Result deal: Quantity = 0.1 lots TradeAction = Buy EntryPrice = 1.10000 ExitPrice = 1.10100 Example #2: Netting account, EUR/USD: 1. Buy 0.1 lots at 1.10000 2. Buy 0.1 lots at 1.10010 3. Buy 0.1 lots at 1.10020 4. Close 0.1 lots at 1.10030 Result deal: Quantity = 0.1 lots TradeAction = Buy EntryPrice = 1.10005 (average price of 1.10000 and 1.10010) ExitPrice = 1.10030
