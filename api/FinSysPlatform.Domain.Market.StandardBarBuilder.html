OnBar(DateTime, double, double, double, double, in Quantity)

Merges incoming bar with last data Calling this method is equal to calling OnTick(DateTime, double, in Quantity) for 4 times consequently: for open, high, low and close prices

public bool OnBar(DateTime time, double open, double high, double low, double close, in Quantity volume)

Parameters

time DateTime

low double
close double
volume Quantity

Returns

bool

true if bar was added, and false if bar was updated

Remarks

This method adds a new bar (if time is larger than last bar time) or updates the last bar (if time is equal to the last bar) If time is less than the last bar time, this method throws an exception Pay attention, that incoming time is aligned first according to Timeframe, with exception to Tick - for ticks time comes as is If coming data updates the last existing bar, then the update goes this way:

  • open remains unchanged (even if open is different to last bar open value)
  • high = Max(last high, high)
  • low = Min(last high, low)
  • close is always assigned from close
  • volume increases by volume

Remarks

This method adds a new bar (if time is larger than last bar time) or updates the last bar (if time is equal to the last bar) If time is less than the last bar time, this method throws an exception Pay attention, that incoming time is aligned first according to Timeframe, with exception to Tick - for ticks time comes as is If coming data updates the last existing bar, then the update goes this way:

  • open remains unchanged
  • high = Max(last high, price)
  • low = Min(last high, price)
  • close is always assigned from price
  • volume increases by volume

Also, pay attention that for ticks, there is a special workaround: if coming ticks go with the same time and it's time to create new bar, then the time for the next bar will be last bar time plus 1 tick
