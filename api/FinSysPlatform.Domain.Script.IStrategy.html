AlgoKey

Any unique value which is used to sign orders This is how the strategy finds its own orders among others AlgoKey

string AlgoKey { get; }

Property Value

string

Remarks

By default, the algo key is combined from strategy type name, Instrument and Bars pararameters: PriceType, BarsType So, the same strategies running on different instruments/periods/price types, have different algo keys.

Bars

Events

BarUpdated

Raises when strategy received the current bar update notification

event Action<IStrategy, IBarSeries>? BarUpdated

Event Type

Action<IStrategy, IBarSeries>

Remarks

The event fires asynchronously. The original event: ChangeCompleted. The original event is handled by the strategy only for the primary input Bars and for inputs that are being listened explicitly. The original event is processed asynchronously in separate thread and after processing it re-throws via this event

ExecutionUpdated

Fires when the execution is updated

event ExecutionUpdatedHandler? ExecutionUpdated

Event Type

ExecutionUpdatedHandler

Remarks

The event fires asynchronously. The original event is fired by account and is handled by the strategy only for executions that have the same Instrument Instrument, Instrument. Then the original event is processed asynchronously in separate thread and after processing reit -throws via this event

Fires when the order is updated

event Action<IStrategy, Order>? OrderUpdated

Event Type

Action<IStrategy, Order>

Remarks

The event fires asynchronously. The original event is fired by account and is handled by the strategy only for orders that have the same Instrument Instrument, Instrument. Then the original event is processed asynchronously in separate thread and after processing reit -throws via this event To check if this order was generated by the strategy, compare strategy AlgoKey with order AlgoKey
