Trade

Represents entry+exit pair. Depending on broker, this pair can be virtual (calculated from real executions) For example, netting accounts afford to add new portions to existing position and close position partially etc. So, for netting account deal can consist of several entries and one exit For hedging accounts usually entry+exit pair usually represents real trade Attention: deal is always calculated from exit. Deal can have multiply entries (for netting accounts) but always has one exit

TradeData

Contains information about a trade from the end of the exchange. For the exchange, any trade executed between to persons is a fact of exchanging a specified amount of instrument at specified price. Exchange-side trade info does not contain information about action (TradeAction) because one participant of the trade bought the volume and another one - sold it. Execution to describe the fact of buying/selling from the trader's end Trade to describe the trade from the trader's end (it has a direction: buy or sell)
