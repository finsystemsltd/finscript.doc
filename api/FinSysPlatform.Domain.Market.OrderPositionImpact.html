Defines the rules to a new apply order to the current existing position

public enum OrderPositionImpact

Fields

[Display(Prompt = "A broker will choose the strategy on its own")] ByDefault = 0

Order will be applied according to account settings

[Display(Prompt = "A new position will be subtracted from opposite existing position (if any)")] Fifo = 1

Order must apply to an existing position (also known as netting accounting type)

[Display(Prompt = "A new position will be created separately from opposite existing position")] Hedging = 2

That order must generate new position

Remarks

Mostly, this enumeration defines the way of opening a new position which is opposite to the existing one For example, there is current 'buy' position. New order opens 'sell' position, which is obviously opposite to the existing one So, for Fifo option, an existing position will be subtracted by a new positions, for more details see https://www.investopedia.com/terms/f/fifo.asp. For Hedging option, a new position will be opened without any impact on the existing one
