Class ScriptCalculationUnit

Namespace
FinScript.Code

Base class for any script unit that works with calculations. The script calculation unit has: input data (Input), output (calculated) data (Plots and (AllValueSeries)) and a state (ScriptCalculationState) that defines the current state of the unit. IScriptCalculationUnit

CalculateMode

Determines how often change notification is called for each bar. OnBarClose means once at the close of the bar. OnEachTick means on every single tick. OnPriceChange means once for each price change. If there were two ticks in a row with the same price, the second tick would not trigger change notification . This can improve performance if calculations are only needed when new values are possible.

public CalculateMode CalculateMode { get; protected set; }

Property Value

CalculateMode

Remarks

Inheritor must set calculation mode according to its logic. By default, OnPriceChange is used as the most appropriate mode for the majority of units. This parameter is not accessible to user to change, because calculation mode (usually) depends on nature of the unit

IsDiscontinuousCalculation

When true, each item of series is calculated independently on previous item. By default, it is set to false, which means that N item of any series in unit can be calculated only after N-1 item.

public bool IsDiscontinuousCalculation { get; protected set; }

Property Value

bool

Examples

Consider that the calculation unit has Result series, and someone outside retrieves Result[1000] item When IsDiscontinuousCalculation is false, all items before 1000 must be calculated, but with true value the state of previous items does not make any sense. So, item 999 will be calculated only when someone will try to get its value

Remarks

Before setting the parameter to true, make sure that the logic of your unit allows you to use values without calculating previous values. Many indicators cannot use discontinuous calculation because each their next value depends on previous value For example, any moving average calculation needs continuous calculation

IsLazyCalculation

When true, it means that script does not calculate any values, even if new data came, until someone outside will ask it for new calculated data Set it to false if you want the script recalculate on each input data change

public bool IsLazyCalculation { get; protected set; }

Property Value

bool

Remarks

Pay attention, that by default this parameter is set to true. If you have an indicator without plots, and you put it onto the chart, by default, you will never receive OnCalculate(CalculateContext) event because there will be no need of calculating data. So, if you want to receive OnCalculate(CalculateContext) event in this case you have to set IsLazyCalculation to false

MirrorPrice(double, double, double)

A service method to get the mirrored price priceToMirror against a pivot price point basePrice using a primary instrument Instrument The formula of calculation is: A+(A-B)*scale, where A = basePrice, B = priceToMirror, scale = scale

public double MirrorPrice(double basePrice, double priceToMirror, double scale = 1)

Parameters

basePrice double
priceToMirror double
scale double

Returns

double

A result price is rounded by RoundToTickSize(double) If the primary instrument Instrument is not set, the result value is NaN

Examples

Example 1. Consider the base price is 100, the second price to mirror is 80, scale is 1. Then the mirrored price is 100+(100-80) = 120 Example 2. Consider the base price is 100, the second price to mirror is 80, scale us 5. Then the mirrored price is 100+(100-80)*5 = 200
