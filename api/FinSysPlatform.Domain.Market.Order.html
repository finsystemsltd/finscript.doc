Remarks

Each Order instance is generated by broker adapter while submitting or changing orders Pay attention that all properties of order are read-only and cannot be changed in instance after its creation. That's why Order has no any change events. Once you get the reference to the order object, all its properties will be constant It means that each time order is changing a broker adapter creates a new instance, so each order changed event is accompanied by new instance Also, each order list request may generate new instances of the orders (depending on broker adapter implementation). So, never compare orders by object references. Use LocalId or OrderId instead

Also, pay attention that this class is abstract: you cannot directly create a new instance of Order. Only broker implementation can do it (SubmitOrders(IReadOnlyList<IOrderSubmissionParameters>, OrderSubmissionContext?) for example) And each broker must create its own derived classes to be able to create order instances

Remarks

Pay attention to orderBasicParametersReference and createCopyOfBasicParameters. Unlike Order(IOrderProperties) this constructor can be used for memory usage optimization if createCopyOfBasicParameters is set to false In this case several order instances can share the same basic order parameters. Basic parameters will be referenced without creation of new instance. This constructor must be used cautiously: do not pass here instances that can change a values of their fields later, because it will impact to all order referencing this parameters

AppliedPosition

For netting positions (Netting) this parameter is not necessary, because the position is sole for each instrument But for hedging positions changes can be applied to already existing position, so if you have to modify existing position (instead of creating new one) you have to specify it by this parameter Pay attention that if applied position is set (not null), PositionImpact value does not make any sense Pay also attention, that instrument of position must be equal to instruments of orders Attention: the referred position has the parameters at the moment of order creation. If the position changes afterwards, the order still refers the detached copy of the position
