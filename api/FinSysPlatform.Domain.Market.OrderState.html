Working = 4

Remarks

Usually lifetime of order looks this way:

  1. If order was successfully filled: New -> Submitted -> Accepted -> Working (for pending orders) -> PartiallyFilled (may be skipped) -> Filled
  2. If order was rejected on client side New -> Rejected
  3. If order was rejected on broker side New -> Submitted -> Rejected
  4. If order was cancelled (by user or automatically) New -> Submitted -> Working -> Cancelled
  5. If working order was cancelled by user Working -> CancelSubmitted -> CancelAccepted -> Cancelled
  6. If working order was changed by user Working -> ChangeSubmitted -> ChangeAccepted -> Working

For information. FIX API defines next order states (see https://www.fixtrading.org/online-specification/order-state-changes/): 0 = New. Equals to New 1 = Partially filled. Equals to PartiallyFilled 2 = Filled. Equals to Filled 3 = Done for day. Similar to Cancelled 4 = Canceled. Equals to Cancelled 5 = Replaced(Removed/Replaced). 6 = Pending Cancel(e.g.result of Order Cancel Request). Equals to CancelAccepted 7 = Stopped. Presumably equals Cancelled 8 = Rejected. Equals to Rejected 9 = Suspended. A = Pending New B = Calculated C = Expired. Similar to Cancelled D = Accepted for bidding E = Pending Replace(e.g.result of Order Cancel/Replace Request).
