Remarks

Bar storage is a multi-thread so, always use EnterRead() before starting some complex work with data. For example, this piece of code, may produce a error: if (barStorage.Count>0) barStorage.GetAt(0) The correct version is

using (barStorage.EnterRead())
{
  if (barStorage.Count>0)
     barStorage.GetAt(0);
}

Because of multi-thread, the storage does not support  (it will produce many problems with linq operations), as well as any other collection interfaces:

Remarks

The actions for the multi threaded storage are performed in the following order:

  1. Write and read locks both are acquired
  2. The bar storage performs changes
  3. Write lock is released
  4. Event is raised
  5. Read lock is released Pay attention 2 simultaneous updates first line up in the queue (due to a writer lock) but then may raise in any order, since they are performed in the scope of reader lock only. For example, consider two parallel actions: ItemAdded and Cleared Let's presume they are lined up in order: Add, Clear. After both writer locks are released the following sequence of events is possible: a) ItemAdded and Cleared. This is most likely to happen, because CPU tends to execute the first event first b) Cleared and ItemAdded. This is not likely to happen, though it is possible
