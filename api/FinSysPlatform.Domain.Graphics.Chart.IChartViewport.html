Interface IChartViewport

Namespace
FinSysPlatform.Domain.Graphics.Chart

The view port object is supposed to reflect Y and X axes states and provide calculation methods to transform values to coordinates and vice versa Pay attention that unique instance of the chart view port is created for each X+Y axes (but not for each chart element)

Returns true if Invalidate() method was already called and not still not processed

bool IsInvalidated { get; }

Property Value

bool

Remarks

This property can be used to optimize the drawing process: the whole drawing process looks this way: invalidation request => put into queue => redraw Consequently, if the invalidation request was already performed and is not processed yet, we can skip any next coming invalidation request

MaxValue

The highest displayed value on the chart scale (Y axis)

double MaxValue { get; }

Property Value

double

Remarks

The max value is calculated from all chart elements attached to the view port The view port, on its turn, is always associated with Y axis (generally speaking, it is associated with the both X+Y axes, but considering the chart has only one X axis, in fact we're speaking about Y association only)
