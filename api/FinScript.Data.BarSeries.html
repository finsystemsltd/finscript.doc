Remarks

Notes about environment. The simplest and most efficient way to initialize environment of any script unit is to use SetInput(TSeries) With this case we do not need any extra initialization. Considering that script units can create another script units via standard new() operator and considering that the sole link which connects creator with its creature is input, using input to transmit environment seems good idea

Methods

EnumerateItems()

Returns series as a enumerable

public IEnumerable<SeriesItem<Bar>> EnumerateItems()

Returns

IEnumerable<SeriesItem<Bar>>

Remarks

To avoid confusion, series does not provide explicit support of IEnumerable<T>, neither IReadOnlyCollection<T> or IReadOnlyList<T> First of all, because series is multi-thread collection and any enumeration of the list may lead to errors (collection was changed during enumeration) Use the method cautiously, wrap your enumerations with LockForRead() There are several methods where item access can be performed without locking:

  1. OnCalculate/OnBarUpdate method in scripts
  2. OnStateChanged method in scripts
  3. DataModification events Changed, Changed2)

GetIndex(DateTime, TimeToIndexMappingMode)

Returns the first item that matches the time stamp of the "time" parameter provided.

public int GetIndex(DateTime time, TimeToIndexMappingMode mappingMode = TimeToIndexMappingMode.NearestLeft)

Parameters

time DateTime

Time stamp to be converted to an absolute item index

mappingMode TimeToIndexMappingMode

Defines a mode of matching time

Returns

int

Absolute index of item which has the same time to time If there is not exact matching then

  1. if mappingMode is set to ExactMatching then the method returns -1
  2. otherwise nearest left or right index is returned (depending on mode, see details in TimeToIndexMappingMode) Attention: with NearestRight the method may return index in ranges [0..Count] with NearestLeft the method may return index in ranges [-1..Count-1] Always check result for being in ranges [0..Count -1] !
