Mode to search bar by index. GetIndex(DateTime, TimeToIndexMappingMode)

public enum TimeToIndexMappingMode

Fields

ExactMatching = 0

A time of bar must match exactly to specified time

NearestLeft = 1

If there is no exact matching, then the nearest left bar will be returned For example: specified time: 2020.01.01 18:15:10. This time lays between 2 items: 2020.01.01 18:15:00 and 2020.01.01 18:16:00 In this case the left item (2020.01.01 18:15:00) will be returned because it is implied that the item lasts from 2020.01.01 18:15:00 up to 2020.01.01 18:15:59.9999999999 and consequently it includes the specified time Pay attention that while using this mapping mode you will always get index less than Count For example, even if you ask for 9999.12.31 and your last item has time 2020.01.01 you will get the last index Also, you may get -1 if a specified time is earlier than the first item item in series

NearestRight = 2

If there is no exact matching, then the nearest right bar will be returned For example: specified time: 2020.01.01 18:15:10. This time lays between 2 items: 2020.01.01 18:15:00 and 2020.01.01 18:16:00 In this case the right item (2020.01.01 18:16:00) will be returned because it is greater than 2020.01.01 18:15:00 Pay attention that this mode may return index equal to Count (in other words, index out of real ranges) For example, even if you ask for 9999.12.31 and your last item has time 2020.01.01 you will get Count as a result
