Remarks

How to add new attributes to PositionGridRepresenter.Item and map them to grid columns:

  1. Add new public property. If it relates to price, quantity, money, percentage or enum, use appropriate representers (for example, MoneyRepresenter, QuantityRepresenter, InstrumentRepresenter, etc)
  2. Add FinSysPlatform.NetExtension.Attributes.DisplayPropertyAttribute to the property to make it visible in the grid as a column
  3. Add FinSysPlatform.NetExtension.Attributes.ColumnWidthAttribute or FinSysPlatform.NetExtension.Attributes.ColumnWidthInferenceAttribute to set up the initial column width About column widths: each visual column must have either to be an explicitly defined with (FinSysPlatform.NetExtension.Attributes.ColumnWidthAttribute set for the property) width or an implicitly defined via representer type (MoneyRepresenter for example)

Note: in some tasks, PositionGridRepresenter.Item may be used outside of grid context or for grids with explicitly defined columns. In other words, FinSysPlatform.NetExtension.Representers.Grids.GridRepresenter<T>.ColumnDescriptors is not really used. In this case, declared attributes do not make any sense and may be ignored
