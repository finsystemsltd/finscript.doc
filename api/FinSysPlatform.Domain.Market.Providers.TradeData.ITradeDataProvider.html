Subscribe(Instrument, string, TradeDataEventHandler)

Subscribes a handler to trades data of specified instrument Each subscriber must provide an unique name (see subscriberKey) in the scopes of instrument which then will be used for unsubscribing You can use the same subscription key for different instruments But you cannot call subscription several times with the same subscriberKey but with different handler If you call subscription with the same parameters several times, all repetitions are just ignored If you want to subscribe to different models simultaneously you have to use different subscription keys

void Subscribe(Instrument instrument, string subscriberKey, TradeDataEventHandler handler)

subscriberKey string

Examples

Correct subscriptions: "MySubscription","LTCBTC", handler "MySubscription","ETHBTC", handler Incorrect subscriptions: "MySubscription","LTCBTC", handler "MySubscription","LTCBTC", handler
