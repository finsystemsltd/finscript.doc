The container to associated a set of units with "parent" unit In fact, there is no any link between parent and attendant on logic level: both units live separately and even may have different inputs The association makes sense only for a chart: having this association, the chart can add attendant units along with parent unit and then remove them at once Pay attention that attendant units feature is accessible for chart-based calculation unit, which means there is no any relation on IScriptCalculationUnit level. So, attendant units are not synchronized with parent in anyway. Particularly, there are no automatic actions: 1. They do no get the input Input of the parent automatically 2. They do not calculate their values along with master 3. They dot not change their states along with master 4. If a superior unit is being disposed, its attendant units are not disposed with it 5. If attendant unit is being disposed, it does not detach from superior unit automatically

On the other hand, there is automation on the chart's end. A chart uses the next logic 1. When the unit is being added to the chart, all its attendant units are also will be added 2. When the unit is being removed from the chart, all its attendant units are also will be removed
