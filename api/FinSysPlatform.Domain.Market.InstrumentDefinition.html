PointSize

Indicates the minimal possible fraction of the price. Usually it is mirror of Precision For example, it Precision=2, Point size most likely will be 0.01 Do not confuse with TickSize, which describes the minimal possible price change. For example, point size 0.01, tick size = 0.05, which means that minimal price change is 5 points (pips)

MirrorPrice(double, double, double)

A service method to get the mirrored price priceToMirror against a pivot price point basePrice The formula of calculation is: A+(A-B)*scale, where A = basePrice, B = priceToMirror, scale = scale

public double MirrorPrice(double basePrice, double priceToMirror, double scale = 1)

Parameters

basePrice double

A pivot point to mirror the second point

priceToMirror double

A point to mirror

scale double

Returns

double

A result price is rounded by RoundToTickSize(double)

Examples

Example 1. Consider the base price is 100, the second price to mirror is 80, scale is 1. Then the mirrored price is 100+(100-80) = 120 Example 2. Consider the base price is 100, the second price to mirror is 80, scale us 5. Then the mirrored price is 100+(100-80)*5 = 200

RoundDownToTickSize(double)

Rounds down to the nearest tick. For example, ES 03-14 price of 1824.93 will return value of 1824.75

public double RoundDownToTickSize(double price)

Parameters

price double

Returns

double

Remarks

Do not confuse PointSize and TickSize Point size is a minimal fraction of price, tick size is a minimal price change. Tick size can be equal or greater than point size, but never less This method works with TickSize! Usually, the price rounded to tick is needed to setup order, because broker requires input prices aligned to tick
